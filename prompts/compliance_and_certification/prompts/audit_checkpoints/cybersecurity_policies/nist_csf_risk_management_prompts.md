# NIST Cybersecurity Framework (CSF) and Risk Management Audit Checkpoint Prompts

## Overview
This collection provides comprehensive audit checkpoint prompts for the NIST Cybersecurity Framework (CSF) 2.0 and enterprise risk management frameworks including NIST RMF, ISO 31000, and integrated threat modeling approaches.

---

## NIST CSF 2.0 Core Functions

### 1. GOVERN (GV) - Cybersecurity Governance and Management

#### GV.OC: Organizational Context
```
PROMPT: Assess our organization's cybersecurity governance structure and mission alignment.

Analyze our enterprise cybersecurity governance against NIST CSF 2.0 GOVERN function:

ORGANIZATIONAL CONTEXT ASSESSMENT:
• Mission Integration: How well is cybersecurity integrated into organizational mission, objectives, and activities?
• Stakeholder Understanding: Do all stakeholders understand their cybersecurity roles and the organization's risk tolerance?
• Legal/Regulatory Environment: How completely do we understand applicable cybersecurity legal and regulatory requirements?
• Critical Technology Assessment: Are we accurately identifying and managing critical technology and systems?

GOVERNANCE STRUCTURE EVALUATION:
• Board/Executive Oversight: What is the maturity level of cybersecurity oversight at board and executive levels?
• Risk Management Integration: How effectively is cybersecurity risk integrated into enterprise risk management?
• Performance Measurement: Do we have appropriate cybersecurity performance metrics and reporting mechanisms?
• Continuous Improvement: Is there a systematic approach to cybersecurity governance improvement?

OUTPUT REQUIREMENTS:
1. Governance maturity assessment (Initial/Developing/Defined/Advanced/Optimizing)
2. Critical gaps in organizational context understanding
3. Stakeholder role clarification needs
4. Integration opportunities with enterprise risk management
5. Specific recommendations for governance enhancement

Include specific examples and evidence for each assessment area.
```

## NIST Risk Management Framework (RMF) Integration

### RMF Step 1: Categorize Information Systems
```
PROMPT: Conduct NIST RMF system categorization and security control baseline selection.

Perform comprehensive system categorization per NIST SP 800-60 and FIPS 199:

SYSTEM CATEGORIZATION ANALYSIS:
• Information Types: What types of information does the system process, store, or transmit?
• Security Objectives Impact: What is the potential impact on confidentiality, integrity, and availability for each information type?
• System Boundary Definition: Is the system boundary clearly defined and documented?
• System Interconnections: Are all system interconnections identified and their security implications assessed?

SECURITY CONTROL BASELINE SELECTION:
• Baseline Selection: Which NIST SP 800-53 baseline applies (Low, Moderate, High) based on system categorization?
• Control Tailoring: Are security controls appropriately tailored for the specific system and environment?
• Control Supplementation: Are additional controls needed beyond the baseline requirements?
• Compensating Controls: Are compensating controls identified where baseline controls cannot be implemented?

CATEGORIZATION DOCUMENTATION:
• Security Categorization Report: Is system categorization properly documented per NIST requirements?
• Categorization Rationale: Is the rationale for categorization decisions clearly documented?
• Stakeholder Review: Has system categorization been reviewed and approved by appropriate stakeholders?
• Categorization Updates: Are there processes for updating categorization as systems change?

OUTPUT REQUIREMENTS:
1. Complete system categorization analysis
2. Security control baseline identification
3. Control tailoring recommendations
4. Categorization documentation assessment
5. Specific recommendations for categorization improvement

Include detailed impact analysis and recommended security control baseline for each system.
```

## Integration Requirements
Each prompt should be used in conjunction with existing compliance framework prompts to ensure:
- Cross-framework compliance alignment
- Integrated risk management across all regulatory requirements
- Consistent security control implementation
- Unified incident response and recovery procedures
- Comprehensive audit trail and documentation

---

## Usage Guidelines
1. **Risk-Based Prioritization**: Focus on highest-risk areas identified through risk assessment
2. **Stakeholder Engagement**: Include appropriate stakeholders in assessment and planning activities
3. **Continuous Monitoring**: Implement ongoing monitoring for all assessed areas
4. **Documentation Standards**: Maintain comprehensive documentation per NIST and ISO standards
5. **Integration Planning**: Ensure integration with existing compliance and security programs

---

## Output Standards
All assessments should provide:
- Current state maturity ratings
- Gap analysis with specific findings
- Risk-prioritized recommendations
- Implementation roadmaps with timelines
- Success metrics and monitoring procedures
- Integration points with other compliance frameworks
